# フォトアルバムアプリ（MVP）

「どれもいい写真すぎて選べない」人のための、選ぶ負担を極限まで減らすフォトアルバム。
**探す → 提案（オススメ順3枚）→ 選ぶ（または全部入れる）→ アルバムにまとめる → 本のようにプレビュー → リンク共有** の一連ループを、ローカル完結・費用ゼロ・オフラインで実現します。

- バックエンド: Python 3.12 + FastAPI
- DB: SQLite（単一ファイル）
- AI: ローカルCLIP（open_clip）※任意。**未導入でもメタ照合フォールバックで全機能が動作します**
- 画像処理: Pillow / OpenCV（EXIF・主要色・ブレ/明るさ）
- フロント: 素のHTML/JS（ビルド不要）

画像は一切外部に送信しません。共有はアカウント不要のトークン付きURLを自前発行します。

## セットアップ・起動手順

### 1. 依存インストール

```bash
cd "C:/Users/me/Desktop/claude start/photo-album-app"
python -m pip install -r requirements.txt
```

必須依存（fastapi / uvicorn / python-multipart / Pillow / numpy / opencv-python-headless）だけで、
心臓部ループはフォールバックモードで完全に動作します。

### 2. （任意）CLIPによる意味検索を有効化

キーワードの「意味的な」検索や、ゼロショット被写体タグ（海／食べ物／夜景…）を使いたい場合のみ、
`requirements.txt` 末尾の2行のコメントを外して追加インストールします。

```bash
python -m pip install torch open_clip_torch
```

- 初回はモデル（数百MB）を自動ダウンロードします。**初回ロードは重い**です。
- 未導入のままでも起動でき、後述のフォールバックで動作します。
- 一時的にCLIPを無効化してフォールバック挙動を確認したい場合は、環境変数
  `PHOTO_ALBUM_DISABLE_CLIP=1` を付けて起動してください。

### 3. サーバ起動

```bash
cd backend
python -m uvicorn app:app --host 127.0.0.1 --port 8000
```

### 4. アクセス

ブラウザで <http://127.0.0.1:8000/> を開きます。

- 画面右上のバッジで AI 状態（CLIP有効／フォールバック）が分かります。
- 「① 写真を取り込む」で手持ちの写真を選択 → 取り込み。
- 「② 写真を探して選ぶ」でキーワード or 期間から候補（オススメ順3枚）を出し、
  1枚ずつ選ぶ or「3枚まとめて全部入れる」。
- 「③ 本のようにプレビュー & 共有」で見開き表示 → 共有リンク発行。
  発行されたURL（`/share.html?token=...`）はアカウント不要で閲覧できます。

### 動作確認用のサンプル写真（任意）

EXIF撮影日付きのダミー写真を生成できます（本番機能ではなく確認補助）。

```bash
cd backend
python make_samples.py ../sample_photos
```

生成された `sample_photos/*.jpg` を①で取り込むと、海／夜景／食べ物などのキーワードや
期間検索の挙動を確認できます。

## 実装したMUST機能（要件との対応）

| 要件(MUST) | 実装 |
| --- | --- |
| 3.0 取り込み時の自動メタ付与（撮影日/場所/色/被写体タグ） | `ingest.py` + `metadata.py`（EXIF・GPS・主要色・ブレ/明るさ）、被写体タグ（CLIPゼロショット or 簡易タグ） |
| 3.1-1 キーワード検索（AI判断・オススメ順3枚・**順位表示**） | `POST /search/keyword` + `search.py`。各候補に rank（1,2,3）と一致度を返却、UIで順位バッジ表示 |
| 3.1-2 期間選択（日/月粒度・**空欄でも「この月はどう？」**） | `GET /search/period` + `GET /search/period/suggest`（写真が最多の月を提案） |
| 3.2 1枚ずつ選ぶ + **「3枚まとめて全部入れる」逃げ道** | UIの pick ボタン + all-in ボタン → `POST /albums/{id}/photos`（1枚/一括両対応） |
| 3.3 自動配置 + **本のようにプレビュー** | `album.py`（縦横比で single/duo 自動レイアウト）+ 見開きビュー（前へ/次へ） |
| 3.4 **アカウント不要の共有リンク** | `POST /albums/{id}/share`（推測困難なトークン）+ `GET /share/{token}`（読み取り専用）+ `share.html` |
| 4.1 被写体タグ中心から開始 | 候補ラベル群 `config.SUBJECT_LABELS`（拡張可能） |
| 4.2 オススメ順=一致度+ブレ/明るさ | `search.py` の total = w1*match + w2*quality（重みは `config.py`） |
| 6章 心臓部ループ通し / 7章 手を止めさせない設計 | 逃げ道ボタン・空欄期間提案・取り込み進捗表示を実装。CLIP有無に関わらずE2E成立 |

## フォールバック方針（モデル未取得時）

CLIP（torch/open_clip）が未導入・ロード失敗でも、心臓部ループは全て動きます（基本設計10章）。

| 状態 | 挙動 |
| --- | --- |
| CLIP有効 | キーワードを意味的類似（コサイン）で検索。被写体タグをゼロショット付与。 |
| CLIP未導入/失敗（既定） | 埋め込み生成をスキップ。キーワードは **被写体タグ・主要色・簡易タグとの語一致** で代替。品質チェック（ブレ/明るさ）は常時有効。逃げ道・期間提案・見開き・共有は通常動作。 |

- API形状・画面遷移はどちらでも同一（フロントはフォールバックを意識しません）。
- EXIF・主要色・ブレ/明るさは常にモデル非依存で算出されます。

## 既知の制約・未対応（本MVPスコープ外）

- 紙フォトブック出力（WANT）／色フォルダ分け（LATER）／日めくりカレンダー（LATER）は未実装。
  ただしデータ設計で `width/height/aspect_ratio`・`dominant_colors` を保持し将来拡張の布石済み。
- 好み学習によるオススメ順の高度化・構図評価は未実装（項目・重みの拡張余地のみ）。
- **編集系APIは認証なし**（ローカル利用前提）。共有ビューのみトークン検証。外部公開時は要追加（TODO）。
- 共有は同一サーバ上での解決を前提。インターネット越しの外部公開はスコープ外
  （トークン方式はそのまま外部公開へ流用可能）。
- 埋め込み検索は全件コサイン類似度計算。写真枚数が非常に多いと遅くなりうる（MVP規模では許容）。
- フォールバック時のキーワード一致は語ベースのため、CLIP有効時より粗くなります（挙動差）。

```

├─ README.md               … 本書
├─ requirements.txt        … 依存（必須 + 任意のCLIP）
├─ docs/basic-design.md    … 基本設計書
├─ backend/
│  ├─ app.py               … FastAPIアプリ（全API + フロント配信）
│  ├─ config.py            … 設定（パス・重み・候補ラベル・同義語）
│  ├─ db.py                … SQLite永続化層
│  ├─ ingest.py            … 取り込みパイプライン（保存/サムネ/メタ/埋め込み）
│  ├─ metadata.py          … EXIF・主要色・ブレ/明るさ・簡易タグ算出
│  ├─ clip_engine.py       … CLIP埋め込み（フォールバック付き）
│  ├─ search.py            … 検索・オススメ順スコアリング
│  ├─ album.py             … 自動ページ配置

├─ frontend/
│  ├─ index.html           … 編集用SPA（取り込み/検索/選択/プレビュー/共有）
│  ├─ share.html           … 共有ビュー（読み取り専用）
│  ├─ app.js               … フロント制御
│  └─ style.css            … スタイル
└─ storage/                … 実行時生成（DB・原本・サムネ。.gitignore対象）
```
