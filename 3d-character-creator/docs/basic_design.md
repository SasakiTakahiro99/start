# 3Dキャラクタークリエイター MVP 基本設計書

## 1. 目的・概要

ブラウザ上で動作する、パーツごとにカスタマイズ可能な3Dキャラクター作成アプリのMVPを開発する。
ユーザーは目・鼻・口・輪郭・体形・肌の色などのパラメータをスライダーやカラーピッカーで調整して
「自分好みの3Dキャラクター」を作成し、作成したキャラクターを小さなフィールド上でWASD/矢印キー操作により
歩かせて確認できる。作成したキャラクターのパラメータは`localStorage`に保存し、リロード後も復元される。

### 要件サマリ
- Three.js（CDN、ビルドステップなし）を用いたブラウザ単体動作
- リアルな人間ではなく、Mii/VRChat簡易アバター的な「滑らかでスタイライズドされた頭身」
- 外部3Dモデルファイル（.glb/.gltf/.fbx等）・画像素材は一切使用せず、プリミティブ形状の組み合わせと
  パラメータ変形のみでキャラクターを構築する
- カスタマイズ対象: 性別（体型プリセット）、目、鼻、口、輪郭・顔の形、体形、肌の色、（余裕があれば）髪型・髪色
- 小さな地面フィールド上でのキャラクターの平面移動（三人称追従カメラ）
- キャラクターパラメータのJSON化・`localStorage`保存/復元

### 想定案件規模
MVP・試運転案件のため、設計書は簡潔さを優先し、後工程（詳細設計・実装・テスト）が迷わない最低限の
情報に留める。過剰な作り込み（アニメーションリグ、物理演算、マルチキャラクター管理等）は行わない。

## 2. ディレクトリ構成

既存のPDF編集ツール一式（リポジトリルート直下の`index.html`, `api_server.py`等）とは完全に独立させ、
新規ディレクトリ`3d-character-creator/`配下に完結させる。

```
3d-character-creator/
├── docs/
│   └── basic_design.md        # 本ドキュメント（以後、詳細設計書・テスト仕様書等もここに追加）
├── index.html                 # エントリポイント。DOM構造・UIレイアウト・style・Three.jsのCDN読込
├── js/
│   ├── main.js                # 初期化・レンダリングループ・シーン/カメラ/レンダラー管理
│   ├── character.js            # プリミティブ形状からキャラクターモデルを構築・パラメータ反映
│   ├── field.js                 # 地面フィールド（平面・簡易な区画/模様）の生成
│   ├── controls.js             # キーボード入力によるキャラクター移動・追従カメラ制御
│   ├── customization-ui.js     # スライダー/カラーピッカー/トグルの生成とイベントバインド
│   └── storage.js              # localStorageへの保存・読込・パラメータのシリアライズ
└── css/
    └── style.css                # UIレイアウト（キャンバス領域・カスタマイズパネル）のスタイル
```

補足:
- ビルドステップは設けない。全てのJSファイルは`<script>`タグ（`type="module"`推奨）で`index.html`から
  直接読み込む素朴な構成とする。
- サーバーサイドの新規実装は行わない。ローカル確認時にブラウザのファイル直開き（`file://`）で
  CORS等の制約が出る場合のみ、Python標準ライブラリ`http.server`をREADME等に案内する補助手段として
  想定する（本アプリの機能としては実装しない）。
- 外部3Dモデルファイル・画像素材は、ダウンロードも同梱も行わない。

## 3. 画面構成・UI概要

単一画面（1ページ）構成。画面は大きく「3Dキャンバス領域」と「カスタマイズパネル」の2ブロックに分割する。

```
+--------------------------------------------------------------+
| ヘッダー: タイトル / 保存状態表示 / 「保存」「リセット」ボタン |
+---------------------------------------+----------------------+
|                                       |  カスタマイズパネル   |
|                                       |  ------------------  |
|                                       |  [性別] トグル/セレクト |
|         3Dキャンバス領域              |  [体形] 身長/肩幅/体重感 |
|         （Three.jsレンダラー）         |         各スライダー   |
|         キャラクター + 地面フィールド   |  [肌の色] カラーピッカー |
|         三人称追従カメラ               |  [目] 大きさ/間隔/色    |
|                                       |  [鼻] 大きさ/形状       |
|                                       |  [口] 大きさ/形状       |
|                                       |  [輪郭] 顔の形セレクト  |
|                                       |  [髪型/髪色]（余裕あれば）|
|                                       |                       |
+---------------------------------------+----------------------+
| フッター: 操作説明（WASD/矢印キーで移動 の案内テキスト）        |
+--------------------------------------------------------------+
```

- レイアウトはPC画面（マウス+キーボード操作）を主眼としたシンプルな2カラム構成（左: キャンバス、右: パネル）。
  レスポンシブ対応・モバイル操作は本MVPの対象外とする。
- カスタマイズパネルの各コントロールを操作すると、即座に3Dキャンバス上のキャラクターへ反映される
  （リアルタイムプレビュー）。
- 「保存」ボタン押下でlocalStorageへ保存。ページロード時に保存データがあれば自動的に読み込み反映する。
  「リセット」ボタンでデフォルトパラメータへ戻す（保存データの削除を伴うかは詳細設計で確定）。

## 4. 機能一覧

| # | 機能 | 概要 |
|---|------|------|
| 1 | キャラクター生成 | プリミティブ形状（球・カプセル・円柱等）を組み合わせ、デフォルトパラメータで頭身の整ったスタイライズドキャラクターをシーン上に構築する |
| 2 | パーツカスタマイズ | UIパネルの操作に応じ、キャラクターの各パーツ（体形・肌色・目・鼻・口・輪郭・髪、性別プリセット）の形状/色/位置パラメータをリアルタイムに更新する |
| 3 | フィールド表示 | 小さめの平面地面（芝生風または単色）をシーンに配置し、キャラクターが乗る土台とする |
| 4 | 移動操作 | WASD/矢印キー入力を受け取り、キャラクターを地面平面上で移動・向き変更させる。カメラはキャラクターに追従する三人称視点を維持する |
| 5 | 保存/復元 | 現在のカスタマイズパラメータをJSON化してlocalStorageに保存する。ページロード時に保存データがあれば読み込み、キャラクターに反映して復元する。保存データが無い場合はデフォルトパラメータを使用する |

対応範囲外（本MVPでは扱わない）:
- ジャンプ・しゃがみ等の追加アクション
- 複数キャラクターの管理・切り替え、キャラクターの書き出し/共有機能
- サーバーへの保存・アカウント機能・マルチプレイ
- モバイル/タッチ操作対応
- 表情アニメーション・歩行モーション（簡易な移動アニメーション程度は詳細設計判断に委ねるが必須要件ではない）

## 5. 主要なデータ構造（概要）

キャラクターパラメータは1つのJSONオブジェクトとして表現し、`localStorage`に文字列化して保存する。
詳細な項目名・値域・キー命名は詳細設計書で確定するが、概念的な構造イメージは以下の通り。

```jsonc
{
  "version": 1,                 // データ構造のバージョン（将来の移行用）
  "gender": "female",           // 性別プリセット（体型プリセット切替のキー）
  "body": {
    "height": 1.0,              // 身長スケール（基準比）
    "shoulderWidth": 1.0,       // 肩幅スケール
    "weight": 1.0               // 体重感（太さ）スケール
  },
  "skinColor": "#f2c9a1",       // 肌の色（カラーコード）
  "face": {
    "shape": "round",           // 輪郭・顔の形（プリセット種別）
    "eyes": {
      "size": 1.0,
      "spacing": 1.0,
      "color": "#3b2a1a"
    },
    "nose": {
      "size": 1.0,
      "shape": "normal"
    },
    "mouth": {
      "size": 1.0,
      "shape": "normal"
    }
  },
  "hair": {                     // 余裕があれば実装。無い場合は固定/非表示扱い
    "style": "short",
    "color": "#222222"
  }
}
```

- 保存対象はこのパラメータオブジェクトのみ（位置・向きなどの実行時の動的状態は保存しない。
  リロード時は常にフィールド上の初期位置からスタートする）。
- パラメータの正規化・値域チェック・デフォルト値定義は詳細設計書で確定する。

## 6. 外部インターフェース

- **Three.js**: CDN経由で`<script>`読み込みする。ビルド・パッケージマネージャは使用しない。
  - 読み込み方針: 安定版のES Modules形式CDN（例: `unpkg`または`jsdelivr`が提供する`three@<バージョン>/build/three.module.js`、
    および補助が必要な場合は`examples/jsm/`配下のアドオン）を用いる。具体的なバージョン番号・CDN URLの確定と
    import mapの要否は詳細設計書で決定する（本設計書では「CDN経由・ビルド不要」という方針のみ規定）。
  - 追加のThree.js公式アドオン（軌道カメラ制御等）を使う場合も、npm等の依存追加は行わず、CDNから同様に
    読み込む方針とする。
- 上記以外の外部API・バックエンド通信は存在しない（本アプリはネットワーク通信不要で完結する）。

## 7. 非機能要件・制約

- **ビルド不要**: npm/webpack等のビルドツールチェーンを導入しない。HTML/CSS/JSファイルをブラウザで
  直接（または簡易な静的サーバー経由で）開けば動作すること。
- **外部アセット禁止**: 3Dモデルファイル（.glb/.gltf/.fbx等）・テクスチャ画像・外部フォント等の
  ダウンロード・同梱を行わない。キャラクターの形状は全てThree.jsのプリミティブジオメトリ
  （球・カプセル・円柱・直方体等）とパラメトリックな拡大縮小・変形のみで構築する。
- **ブラウザのみで完結**: サーバーサイドの新規実装は行わない。永続化はブラウザの`localStorage`のみを使う。
- **対象ブラウザ**: 最新版のChrome/Edge（Windows）を主対象とする。WebGL対応が前提。
  他ブラウザでの動作保証・後方互換対応は本MVPの対象外とする。
- **既存プロジェクトとの独立性**: 本アプリのファイルは全て`3d-character-creator/`配下に配置し、
  リポジトリルート直下の既存PDF編集ツール関連ファイル（`index.html`, `api_server.py`, `pdf_editor.py`等）
  には一切手を加えない・混在させない。
- **パフォーマンス**: MVPとして高負荷な描画（大量ポリゴン・高解像度テクスチャ・複雑な物理演算）は
  想定しない。一般的なノートPC相当の環境でスムーズに動作する軽量な構成とする。
- **国際化・アクセシビリティ**: 画面文言は日本語のみを想定し、多言語対応は行わない。

## 8. 後工程への申し送り事項

- 詳細設計書では以下を確定させること:
  - Three.jsの具体的なバージョン・CDN URL（import mapの要否含む）
  - キャラクターパーツの具体的なジオメトリ構成（各パーツをどのプリミティブ形状の組み合わせで表現するか）
  - パラメータの値域・デフォルト値・体型プリセット（性別）ごとの差分の定義
  - カメラ追従の具体的な挙動（追従距離・角度・スムージングの有無）
  - localStorageのキー名・保存タイミング（自動保存か明示的な保存ボタンのみか）・エラー時のフォールバック
  - 髪型・髪色の対応要否（余裕次第で実装/簡略化を判断）
